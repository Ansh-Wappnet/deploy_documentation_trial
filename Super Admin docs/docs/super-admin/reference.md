# Reference

## Glossary

| Term | Definition |
|------|------------|
| **Client** | A tenant organization using the Recon AI platform. Data is siloed per client. |
| **Domain** | A logical subdivision or entry point associated with a client. Used for routing. |
| **Module** | A functional capability of the system (e.g., Reconciliation, Reporting). |
| **Permission** | A granular rule controlling access (read, write, etc.) to specific modules. |
| **Rule Component** | A reusable unit of logic used to construct reconciliation rules. |
| **Subscription Plan** | A tiered package defining feature access, usage limits, and pricing. |
| **Schema** | The database namespace isolated for a specific client. |

## Appendix

### Future Enhancements (Roadmap)
*These features are planned for upcoming releases:*

- **Advanced Audit Logging:** Deeper insights into specific field-level changes.
- **Admin Analytics Dashboard:** Usage statistics for the platform (e.g., total volume processed across all clients).
- **Role-Scoped Admin Views:** Restricted admin access (e.g., "Support User" who can view but not edit).
- **Bulk Management:** Tools to update multiple clients or rules simultaneously.
- **Approval Workflows:** Multi-step approval processes for critical changes.
